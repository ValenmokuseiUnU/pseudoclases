Pseudo clases

Una pseudoclase es un selector que marca los elementos que están en un estado específico, por ejemplo, los que son el primer elemento de su tipo, o aquellos por los que el cursor les pasa por encima.

:active

La pseudoclase :active especifica cuando un elemento está activo, es decir, cuando está siendo clicado por el usuario, de esta manera podemos ver cuando el boton se presiona y cambia

Ejemplo=


:checked

El selector checked es ideal para cambiar de estilo a las checkbox que son las casillas de verificación en los formularios de una manera sencilla sin emplear JavaScript y que funcionan muy bien al emplear las transiciones para los cambios de un estado a otro como hemos visto en otros tutoriales en los que hablamos de las transiciones en CSS

:default

pseudo-clase aplica estilo para formar elementos que se establecen de forma predeterminada en un grupo de elementos similares. Por ejemplo, podemos tener dos botones para enviar datos de formulario al servidor. Uno de estos botones está configurado por defecto, podemos cambiar su estilo a través de :default







:disabled

s una pseudo-clase que representa cualquier elemento deshabilitado. Funciona como una condicional en la que aplica los estilos, siempre y cuando los elementos cumplan. Es decir, cuando los elementos tienen el atributo disabled



:visited

representa enlaces que el usuario ya ha visitado. Por motivos de privacidad, los estilos que se pueden modificar con este selector son muy limitados

BIENVENIDO!

:valid

La pseudoclase :valid de CSS representa cualquier elemento de tipo input u otro elemento del formulario cuyo contenido se valide satisfactoriamente. Esto permite que los campos válidos adopten fácilmente una apariencia que ayuda al usuario a confirmar que sus datos están formateados correctamente.

:enabled

:enabled CSS representa cualquier elemento habilitado. Un elemento está habilitado si puede ser activado (es decir seleccionado, se puede hacer click en él, acepta que se le introduzca texto, etc.) o si accepta el foco. El elemento también tiene un estado deshabilitado el en cuál, no puede ser activado ni acepta el foco







:first-child / :last-child

Representan el primer y último elemento entre un grupo de elementos hermanos, respectivamente.

:focus

Representa un elemento (como una entrada de formulario) que ha recibido el foco. Generalmente se activa cuando el usuario hace clic o toca un elemento.


:hover

Coincide cuando el usuario interactúa con un elemento con un dispositivo señalador, pero no necesariamente lo activa. Generalmente se activa cuando el usuario se desplaza sobre un elemento con el cursor.
